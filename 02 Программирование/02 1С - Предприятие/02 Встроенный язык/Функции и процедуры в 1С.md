
Процедуры — это именованные операции, предназначенные для многократного использования.

Функции -- это процедуры, которые возвращают результат своей работы

Процедуры и функции можно объединить общим названием **методы**

```
// Процедура может быть с параметрами и без них
Процедура ПечатьСуммы(Число1, Число2) Экспорт

    Сумма = Число1 + Число2;
    Сообщить("Сумма чисел: " + Сумма);

КонецПроцедуры

// Вызов процедур
ПечатьСообщения();
ПечатьСуммы(5, 10);
```

Модификатор `Экспорт` делает метод открытым для использования другими модулями

**Функции**

```
// Функция с параметрами
// 2-й параметр имеет значение по умолчанию
Функция УмножитьЧисла(Число1, Число2 = 10)

    Возврат Число1 * Число2;

КонецФункции
```

**Модификаторы параметров**

```
// передача числа по значению (с копированием)
Процедура УвеличитьНаДесять(Знач Число)

    Число = Число + 10;
    Сообщить("Внутри процедуры: " + Число);

КонецПроцедуры

// передача числа по ссылке
Процедура УвеличитьНаДесять(Число)

    Число = Число + 10;
    Сообщить("Внутри процедуры: " + Число);

КонецПроцедуры
```

**Особенности методов в 1С:**

- Ссылочные типы, кроме строки, независимо от модификатора параметра передаются по ссылке. Это такие типы, как ...

- По умолчанию параметры передаются **по ссылке**

- 1С не поддерживает перегрузок методов (отсутствует ad-hoc полиморфизм). Встроенный язык 1С:Элемент поддерживает перегрузку методов

- 1С не поддерживает функциональную парадигму. Типов, предсталяющих методы как объекты не существует в 1С. Соответственно и синтаксиса лямбда-выражений тоже нет
